# 会员卡代码迁移总结

------
最近在修改会员卡购买项目的代码，然后想到了一些事情

- 刚进公司不久，经理口头叫我去迁移会员卡的代码，然后就花点个人时间去做，由于对平台逻辑不是非常熟悉，仅仅是换一种方式把代码搬迁到service下，所以是搬迁不是重构；后来主管叫我重构哦会员卡的代码，优化效率问题，那么问题来了.....

> * 有时候你觉得自己做得不错，那是站在自己的角度；如果站到领导的角度去评价，你会发现很尴尬；

> * 一致性、冗余性、结构性等等都没有考虑到，直到主管点出来了，才意识到尴尬，难怪说我不用心；

- 做得不好被骂被说才是正常的反应，至少意识到自己在那些方面做得不足，然后下次注意并改正；

- 要敢于去做，初生牛犊不怕虎，怕说怕骂是很难进步的；

- 要渐渐的懂得会举一反三
